HTML

  • The head element contains metadata. Data info about data that the user does not see.
  • The body element contains visible content shown to the user.
  • Tags refer to what appears inside angle brackets.
  • Element refers to entire element from opening to closing tage.
  • The parent element is the element that holds all of the children.
  • The child elements refers to elements nested inside other elements.
  • Best practice is to use semantic HTML elements whenever possible. The body element is an example of semantic HTML element.
  • src, alt, id and class are examples of attributes.
  • When using headings order matters, and the biggest heading should only appear once on each page.

CSS

  • An external CSS style sheet is the most common way to style a webpage.
  • An external sheet allows developers to keep all their CSS rules in a separate file, which makes design changes easier. This is also referred to as separation of concerns.
  • The selector defines the element or attributes to which the rules/declarations will apply.
  • Then you declare your property and values for the element.
  • Property that we want to apply, like color, and value of that property, like red.
  • Margin property indicates how much space we want around the outside of an element. It isn't visible to the user and acts as a buffer. When it is set to 0 we are saying we want no extra space around the element.
  • Padding property adds space around the content inside an element. When you increase padding, you are making the entire element bigger. It is transparent.
  • The CSS Box Model thinks of every html element as a box.
  • The asterick * is used to apply a rule to all elements visible on a page.
  • Two or more elements that will share the same styling can be grouped together using a comma. This helps coders write DRY code.
  • Use the link element to link html with the css sheet.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning a rule to a class attribute with a class selector. The class attribute can be assigned to the element in html.
  • A design guide/system or brand book contains detailed guidelines that detail the colors, fonts, spacing between items on a page, appropriate writing style, etc. for a project.

Git

  • git status: checks what branch I am currently working on.
  • git checkout main: takes me back to the main branch.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add: tracks changes made in branch.
  • git commit -m "command": to commit files. Write git commit commands as imperative commands in the present tense.
  • git pull origin main: to double check if local branch is in sync with base branch. Look for message: Already up to date.
  • git push origin branch-name: use to push changes from local computer to github.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • A string follows the variable.
  • All strings must be surrounded by quotation marks.
  • To access the strong simply refer to the variable's name.
  • Console.log() is a method to output a message to the web console by adding an argument, i.e., a variable.
  • The .log() portion is a method, a set of instructions that can be executed by a computer. It will output whatever is inside the parenthesis.
  • JS files are linked at the bottom of the body element.
  • Control flow is the order in which a computer executes code in a script. In JS code runs from first statement at the top of the page to last statement at the bottom, ie, linear.
  • Using conditional logic you can control the flow of an application, and the order of code affects the outcome.
  • Use conditional statements and loops to control the order in which the code is executed.
  • Conditional statements are 'if' statements. The code executes if the statement meets the condition, or is true.
  • A statement is considered truthy if it means the conditions, or is a value of 'true.'
  • A value is considered falsy if it is one of the following values: 0, -0, On, "", Undefined, NaN, or Boolean data type with the value of false.
  • We can have more than one condition by using 'else if.'
  • A strict equality operator === checks to see if two values are equal.
  • Use an array if you need to perform a task on 50 or more variables at once.
  • Arrays are not limited to single types of data--they can hold strings, numbers, etc.
  • For an array, the data is wrapped in brackets.
  • To work with individual items in an array, we use indexes. Indexes give each piece of data a location.
  • Arrays are 0-indexed.
  • Use for loops if we have an array that contains 100s or more items.
  • A function is a set of instructions that tells the computer how to perform a task.
  • Functions are not automatically executed. You must call it by name in order to execute it.
  • When we declare a function in the JS code, we are creating a reusable block of code that performs a specific task.
